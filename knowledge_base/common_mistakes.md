# Common Mistakes and Pitfalls in JEE Math

## Algebra Mistakes
- Forgetting that sqrt(x^2) = |x|, not x
- Incorrectly canceling terms: (a+b)/a != 1+b
- Wrong sign when squaring: (a-b)^2 != a^2 - b^2
- Forgetting that log is only defined for positive arguments
- Not checking domain restrictions when solving equations
- Dividing both sides by a variable without checking if it could be zero

## Probability Mistakes
- Not considering whether events are independent or dependent
- Confusing permutations (order matters) with combinations (order doesn't matter)
- Forgetting to account for complementary counting: P(at least one) = 1 - P(none)
- Double counting in union of events
- Misapplying Bayes' theorem by confusing P(A|B) with P(B|A)
- Not accounting for with/without replacement correctly

## Calculus Mistakes
- Forgetting chain rule in composite functions
- Wrong sign in derivative of cos(x): d/dx[cos x] = -sin x
- Not checking continuity before applying L'Hopital's rule
- Forgetting to check endpoints in optimization on closed intervals
- Confusing d/dx[e^(f(x))] = e^(f(x)) * f'(x) with just e^(f(x))
- Not verifying that critical point is actually a max/min (using second derivative test)

## Linear Algebra Mistakes
- Assuming matrix multiplication is commutative (AB != BA)
- Forgetting to check determinant != 0 before finding inverse
- Wrong cofactor signs in determinant expansion
- Not reducing to row echelon form correctly
- Confusing eigenvalues of A with eigenvalues of A^(-1)

## General Problem-Solving Pitfalls
- Not reading the question carefully (what is actually being asked)
- Forgetting units or domain constraints
- Not verifying the answer by substitution
- Skipping edge cases (x = 0, negative values, etc.)
- Arithmetic errors in final computation
- Not simplifying the final answer

## JEE-Specific Tips
- In multiple choice, try plugging in answer choices
- Use dimensional analysis to eliminate wrong options
- Check boundary/extreme cases
- For "which of the following" questions, test each option systematically
- Time management: don't spend too long on one problem
